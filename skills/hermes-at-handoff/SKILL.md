# hermes-at-handoff — 话题交接焊死系统

## 触发条件（自动检测，无需手动调用）

本 Skill 在以下情况下**自动激活**，无需用户说"用这个 skill"：

1. 检测到消息包含 `——@#` → 记录"会话开始"，保存上下文到 handoff.json
2. 检测到消息包含 `@#` → 触发"焊死"，调取历史上下文
3. 检测到 `——@#` 后 1 分钟内出现 `@#` → **焊死效果触发**：提取并呈现上下文

## 核心逻辑

### 第一步：检测标记
- 读取当前会话最近的 N 条消息（由系统注入，或由辅助脚本提取）
- 用正则匹配：`——@#`（开头）和 `@#`（结尾）
- 检查时间戳：两个标记间隔是否 ≤ 60 秒

### 第二步：焊死效果（效果触发）
当检测到有效交接序列（`——@#` + `@#` 在 1 分钟内）：

```
调用 conversation_search，查询：
- 当前 WorkBuddy 会话中涉及的所有话题
- 最近 7 天的对话记录中与当前上下文相关的历史
- 输出格式：按时间倒序，列出 15 条关键上下文线索
```

### 第三步：写入 handoff.json
将当前会话的上下文摘要写入：
`~/.workbuddy/handoff/hermes-handoff.json`

```json
{
  "session_start": "<ISO8601>",
  "handoff_triggered": "<ISO8601>",
  "context_summary": "<当前会话核心摘要>",
  "open_topics": ["<未完成事项>"],
  "pending_decisions": ["<待决策项>"],
  "last_action": "<最后执行的动作>"
}
```

### 第四步：新会话承接
在新会话的**第一条消息**中：
- 读取 `~/.workbuddy/handoff/hermes-handoff.json`
- 如果 `handoff_triggered` 存在且距今 ≤ 1 分钟，自动输出：
  ```
  ——@# [上下文焊死]
  检测到会话交接，上下文如下：
  1. ...
  2. ...
  ...
  15. ...
  ```
- 然后正常承接话题

## 标记规范

| 标记 | 含义 | 触发动作 |
|------|------|----------|
| `——@#` | 会话开头/话题开始 | 记录 session_start，写入 handoff.json |
| `@#` | 会话结尾/话题结束，触发交接 | 搜索历史上下文，写入 handoff |
| `——@#` 后 1 分钟内出现 `@#` | **焊死效果触发** | 完整上下文提取 + 自动承接 |

## 上下文提取规则

- 搜索范围：当前 WorkBuddy 工作区 + Hermes 历史会话
- 输出条数：15 条（可调整）
- 每条包含：时间戳 + 话题摘要 + 关键决策
- 去重 + 按相关性排序

## 依赖工具

- `conversation_search`：搜索历史对话
- `update_memory`：写入持久化上下文
- 文件读写：维护 handoff.json 状态

## 使用场景示例

**场景 1：话题结束，旧窗口 → 新窗口**
```
用户（旧窗口）：@# （话题结束）
AI：正在焊死上下文……已保存到 handoff.json

用户（新窗口）：——@# （新会话开始）
AI：——@# [上下文焊死] 检测到交接，上下文如下：...
     承接话题继续...
```

**场景 2：直接新会话（无旧窗口）**
```
用户：——@# 关于 XX 项目
AI：——@# [上下文焊死] （首次会话，无历史上下文）
     好的，XX 项目，从头开始。
```
